
## 2024-05-18 - [Typed JustHTML options objects]
**Learning:** `JustHTML` and `parseDocument` options destructured default parameters like `options = {}` and used `@ts-expect-error` to suppress compiler warnings instead of explicitly typing the `options` signature.
**Action:** Always define interfaces explicitly like `JustHTMLOptions` and `ParseDocumentOptions` for object parameters (even optional configuration ones), allowing the TS compiler to infer and match properties properly instead of suppressing the errors inside the function body.
## 2026-05-02 - Typed Serializer Options in html5lib_serializer.ts
**Learning:** Destructured or default-assigned options objects (`options = {}`) often have `any` or implicit `any` types in legacy JS-to-TS files, leading to many `@ts-expect-error`s when accessing properties like `options.inject_meta_charset`. These can be safely replaced by explicitly defining a dedicated interface with optional properties (e.g. `interface SerializerOptions { inject_meta_charset?: boolean }`). Also, beware that running `bun run lint` can execute auto-formatters (like oxlint) that unintentionally modify unrelated files.
**Action:** When typing options objects, create an interface with optional properties derived from usage within the function body. Always check `git status` after running lint commands and revert unrelated stylistic changes before requesting a code review or committing.

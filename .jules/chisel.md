## 2024-05-24 - [Class Constructor Defaults Reflection] **Learning:** When typing JavaScript classes that have destructured default parameters (like `{ message = null } = {}`), ensure that the class properties are typed to handle the defaults (e.g. `string | null` instead of just `string`), otherwise valid instantiation scenarios can cause type errors if `strictNullChecks` gets enabled. **Action:** Always cross-reference the class properties with constructor defaults during JS-to-TS migration of objects.

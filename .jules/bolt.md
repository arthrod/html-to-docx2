## 2024-03-09 - [Performance Optimization: justjshtml Object Iteration]
**Learning:** Performance benchmarking in the HTML parsing logic showed that iterating over an element's `attrs` (attributes) map with `Object.entries(attrs)` was resulting in severe intermediate array allocation overhead. Refactoring this into a standard `for...in` loop with `hasOwnProperty` effectively speeds up iterating over attributes during high-frequency tasks such as DOM tree parsing by avoiding the creation of these extra intermediate arrays.
**Action:** Replace `Object.entries()` loops with `for...in` alongside `Object.prototype.hasOwnProperty.call()` checks in hot code paths like HTML/XML parsers, where the iteration speed and reduction of memory allocations directly improve CPU utilization and reduces Garbage Collection spikes.

## 2025-02-28 - Deep cloning static and flat objects on hot paths
**Learning:** `cloneDeep` from `es-toolkit/compat` causes significant performance degradation and memory allocation overhead when used indiscriminately on flat attributes or read-only static objects during XML node generation (hot paths). Specifically, deep cloning flat configuration attributes or read-only constants like `paragraphBordersObject` takes ~600ms for 100k iterations compared to ~6ms with a shallow copy or native iteration.
**Action:** Replace `cloneDeep` with shallow spread (`{ ...attributes }`) for flat objects when only avoiding direct mutation is necessary. For static configuration objects that are read-only, remove cloning entirely and rely on native loops like `for...in` to eliminate expensive object creation overhead.

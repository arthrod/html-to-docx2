## 2024-04-17 - Eliminate cloneDeep and Object.keys().forEach() array allocations
**Learning:** In the `@turbodocx/html-to-docx` builder (`src/helpers/xml-builder.ts` and `src/helpers/html-parser.ts`), functions like `buildRunProperties` were doing deep cloning via `cloneDeep` from `es-toolkit` and iterating using `Object.keys().forEach()`. This resulted in significant performance and memory overhead by allocating intermediate array objects for keys and wrapping callback closures. In most scenarios, attributes are flat objects or don't even need cloning since they are just read.
**Action:** Replace `cloneDeep()` with shallow spread `...` when shallow object replication is sufficient. Replace `Object.keys().forEach()` loop structures with either native `for...in` loops accompanied by `hasOwnProperty` checks or classic indexed `for` loops utilizing a cached `Object.keys()` array. This prevents performance-degrading array allocation overheads and speeds up execution significantly during tree traversals.

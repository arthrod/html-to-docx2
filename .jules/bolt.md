## 2025-05-05 - Avoid deep clones on flat and read-only objects in hot paths
**Learning:** `cloneDeep` is excessively expensive in JavaScript environments. In document generation loops like building DOCX XML runs, even small, flat attribute objects cloned with `cloneDeep` incur massive overhead (up to ~10x-15x slower compared to shallow copies). Furthermore, applying `cloneDeep` to read-only module constants (like `paragraphBordersObject`) just to iterate over them is an extreme anti-pattern resulting in severe garbage collection churn.
**Action:** Replace `cloneDeep` with native shallow spreads (`{ ...attributes }`) for flat mutation protection, and iterate directly over constant definition objects using `for...in` instead of cloning them and using `Object.keys().forEach()`.

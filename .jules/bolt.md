## 2024-05-18 - Replacing Object.entries with O(1) checks
**Learning:** When searching for a specific attribute (like `'type'`) inside an object, replacing `Object.entries(obj)` with `for...in` loops is still an O(n) operation. To properly optimize property lookups in a hot path, you must use direct O(1) property access (e.g., `Object.prototype.hasOwnProperty.call(attrs, 'type')`). Furthermore, avoid using `Object.hasOwn` blindly as it requires Node 16.9+ and may break compatibility in older environments, unless you are certain the target runtime supports ES2022. Lastly, avoid replacing high-performance character-by-character `while` loop parsers with regex `.match()` because regex match results allocate new array objects on every successful match, introducing GC pressure.
**Action:** When searching for a single key, I will use `Object.prototype.hasOwnProperty.call(obj, 'key')`. When evaluating regex replacements in parsing hot paths, I will consider memory allocation overhead.

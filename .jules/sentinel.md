## 2025-02-13 - Fix SSRF bypass via IPv6 in isPrivateOrLocalHost
**Vulnerability:** The `isPrivateOrLocalHost` function relied on exact string matches for `localhost` and `127.0.0.1` but failed to handle IPv6 formats properly. It only checked `[::1]` without covering the unspecified address `[::]`, link-local `[fe80::]`, unique local `[fc00::]`, or IPv4-mapped IPv6 `[::ffff:127.0.0.1]`. This could allow attackers to bypass SSRF protections by supplying local IPs in these formats.
**Learning:** The URL utility failed to comprehensively account for IPv6 formatting variations, assuming checking exact strings would suffice. This is a common pitfall when parsing hosts manually instead of relying on a dedicated parsing library or full IP normalization.
**Prevention:** To prevent SSRF bypasses when validating local/private hostnames, strictly check for IPv6 variations including the unspecified address `[::]`, link-local `[fe80::]`, unique-local `[fc00::]`/`[fd00::]`, and recursively validate the inner IP from IPv4-mapped IPv6 addresses like `[::ffff:127.0.0.1]`. Alternatively, use robust IP normalization utilities before performing heuristic blocklists.

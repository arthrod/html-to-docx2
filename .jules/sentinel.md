## 2025-02-28 - URL Protocol Validation Bypass
**Vulnerability:** In `imageToBase64` and `downloadImageToBase64` functions, external URLs were being parsed for the protocol without explicitly trimming whitespace and wrapping the string construction with a base fallback for erroring properly, but additionally, checking URL schemes (`http:`, `https:`, etc.) was potentially bypassed or causing issues without properly constructing the native URL object cleanly (e.g., SSRF / Scheme abuse risks with `javascript:` or `file:`).
**Learning:** In native web and Node fetching contexts, URL strings must be thoroughly validated against the parsed `new URL(str.trim(), 'http://dummy.base')` construct to ensure that scheme logic (`http:`, `https:`, `data:`, `blob:`) accurately restricts fetches to expected web and blob protocols only, thereby preventing potential LFI (`file://`) and command execution or XSS vectors.
**Prevention:** Always validate external resource locations using the native `URL` class constructor, parse and assert the `.protocol` attribute against an allowlist, and fail-safe using `throw new Error('Invalid URL')` before executing `fetch()`.

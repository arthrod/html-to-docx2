## 2024-03-31 - Local File Inclusion (LFI) / Server-Side Request Forgery (SSRF) via Bun fetch
**Vulnerability:** The image downloading functions (`downloadImage`, `downloadImageToBase64`) used unvalidated absolute URLs in `fetch()` calls. Because Bun's native `fetch` supports `file://` URLs out of the box, an attacker could supply `file:///etc/passwd` to read local server files, causing LFI.
**Learning:** Naive regular expressions or URL string matching to block protocols is error-prone because `fetch` often strips leading whitespace, bypassing simple regex checks. Also, standard `URL` constructors throw errors on relative paths unless a base is supplied.
**Prevention:** Always validate protocols using the native `URL` constructor with a dummy base URL (e.g., `new URL(String(url).trim(), 'http://dummy.base')`). Explicitly allow only safe protocols (`http:`, `https:`, `data:`, `blob:`) and gracefully permit relative URLs by verifying they resolve to the dummy base origin.

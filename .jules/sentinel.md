## 2024-04-06 - SSRF Vulnerability in Image Fetching Utilities
**Vulnerability:** Server-Side Request Forgery (SSRF) and Local File Inclusion (LFI) vulnerabilities existed in the image fetching utilities (`downloadImageToBase64` and `downloadAndCacheImage`). User-provided URLs were passed directly to `fetch()` without any protocol validation, allowing malicious actors to fetch arbitrary internal network resources (e.g. cloud metadata via `http://169.254.169.254/...`) or local files (e.g. `file:///etc/passwd`).
**Learning:** This existed because `fetch` gracefully handles different URL schemes without strictly throwing errors for non-HTTP URLs unless configured to do so. Simple type definitions like `string` for URLs provide no security guarantees against arbitrary protocols.
**Prevention:** Always validate URLs using the native `URL` constructor before calling external APIs like `fetch`. Ensure strict checking of `url.protocol` against an allowed list (e.g. `['http:', 'https:', 'data:', 'blob:']`) and explicitly reject dangerous protocols.

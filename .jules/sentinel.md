## 2024-03-24 - LFI and SSRF via Unvalidated fetch() Calls
**Vulnerability:** The application used `fetch()` directly on user-provided image URLs without validating the protocol. Since Bun's native `fetch` API supports `file://` URLs natively, an attacker could supply a `file:///etc/passwd` or `http://localhost/admin` URL, leading to Local File Inclusion (LFI) or Server-Side Request Forgery (SSRF) during the HTML to DOCX conversion process.
**Learning:** In Bun environments, `fetch` acts as an unrestricted client by default, capable of reading local files and accessing internal network resources if not explicitly guarded. Naive URL parsing or omitting protocol validation completely exposes the application to critical severity data exfiltration.
**Prevention:** Always validate external URLs using a strict allowlist of protocols (e.g., `http:`, `https:`) and ensure the validator blocks private IP ranges and `localhost` for absolute URLs before passing them to `fetch()`. The project provides `isValidUrl` in `src/utils/url.ts` specifically for this purpose, and it must be applied to all URL inputs before fetching.

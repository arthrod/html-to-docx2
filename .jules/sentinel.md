## 2025-03-08 - [Server-Side Request Forgery (SSRF) Risk and URL Validation]
**Vulnerability:** The `downloadImageToBase64` function in `src/utils/image.ts` and `src/utils/image-browser.ts` previously fetched user-provided URLs without validating the protocol. This could permit SSRF or Local File Inclusion (LFI) through protocols like `file://` or `gopher://`. Furthermore, initial attempts to mitigate this via simple string matching (`toLowerCase().startsWith('http://')`) were identified as overly brittle and prone to bypasses (e.g., whitespace).
**Learning:** Functions downloading resources must restrict permitted protocols strictly. Validating URLs using string matching is brittle and bug-prone. Furthermore, strict validation logic must still permit legitimate inputs used by the application, such as `data:` URIs or protocol-relative URLs.
**Prevention:** Always rely on proper URI parsing (e.g., using the `URL` constructor: `new URL(url).protocol`) to validate URL protocols securely. Ensure allowed protocols include necessary schema types like `http:`, `https:`, and `data:` depending on the library's feature support context.

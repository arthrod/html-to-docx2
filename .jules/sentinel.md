## 2024-05-15 - [SSRF Protection in Image Fetching]
**Vulnerability:** Server-Side Request Forgery (SSRF) and Local File Inclusion (LFI) in URL parsing for image downloads. Node's `URL` normalizes decimal IP addresses like `http://2130706433/` to `http://127.0.0.1/`, which bypassed naive URL matching, and allowed fetching `file:///` protocols if not strictly validated.
**Learning:** Checking for `localhost` and standard `127.0.0.1` strings is insufficient because URL parsers handle octal and decimal IP representations. Furthermore, local and private networks can be accessed without strict allowlisting of protocols and denying of private IPv4 subnets.
**Prevention:** Always validate protocols using an allowlist (`http:`, `https:`, `data:`, `blob:`). Use `new URL` to parse strings, then evaluate `url.hostname` against explicit checks for private network IP address ranges (`10.x`, `192.168.x`, `169.254.x`, `172.16.x`-`172.31.x`) and localhosts (`localhost`, `[::1]`, `127.0.0.1` and variants) after parsing.

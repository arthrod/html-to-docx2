## 2024-04-10 - [LFI/SSRF via unsanitized fetch in image downloads]

**Vulnerability:** The application was passing user-provided image URLs directly into the native `fetch()` function in `src/utils/image.ts`, `src/utils/image-browser.ts`, and `src/utils/image-to-base64.ts` without validating the protocol. This allowed Local File Inclusion (LFI) via the `file://` protocol or internal network requests (SSRF) via arbitrary unsupported protocols like `gopher://` or `dict://` if supported by the underlying fetch implementation.
**Learning:** Naive regex validation or assuming `fetch()` is safe by default is incorrect. `fetch()` can handle multiple protocols natively depending on the runtime (e.g. Node vs Browser vs Bun). Also, simply validating URLs within a generic `try-catch` block can swallow important protocol violation errors and mask them as generic "Invalid URL" errors.
**Prevention:** Always validate URLs using the native `URL` constructor (e.g., `new URL(String(url).trim(), 'http://dummy.base')`). Explicitly check the `protocol` property against an allowlist of safe protocols (e.g., `http:`, `https:`, `data:`, `blob:`) outside of the try-catch block before making the network request.

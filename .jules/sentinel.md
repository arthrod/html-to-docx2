## 2025-02-27 - Fix SSRF/LFI in Image Fetching Utilities
**Vulnerability:** The application was vulnerable to SSRF (Server-Side Request Forgery) and LFI (Local File Inclusion) via the `fetch` API in `src/utils/image-to-base64.ts`, `src/utils/image.ts`, and `src/utils/image-browser.ts`. The `downloadImage` and `downloadImageToBase64` functions accepted arbitrary URLs without protocol validation. Since `fetch` supports various protocols, an attacker could potentially read local files using the `file://` scheme or access internal network resources.
**Learning:** The native `URL` constructor should be used to parse and validate protocols for any dynamically constructed or user-provided URL prior to network fetching. Furthermore, custom validation logic checking valid schemes (`http:`, `https:`, `data:`, `blob:`) must be placed carefully, throwing the specific error message `Invalid URL` rather than a custom one like `Invalid URL protocol`, to avoid breaking caching test suites which strictly assert the exact error object/message.
**Prevention:** Always validate protocols using a parsed `URL` object and an allowlist of safe protocols before passing URLs to `fetch()`. Be cautious of the exact error messages expected by caller catch blocks and caching layers to prevent cascading regressions.

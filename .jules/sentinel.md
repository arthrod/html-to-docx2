## 2025-04-12 - Prevent SSRF and LFI in Image Fetching
**Vulnerability:** The application used `fetch()` directly with user-provided or document-provided URLs in `src/utils/image-to-base64.ts`, `src/utils/image.ts`, and `src/utils/image-browser.ts` without validating the protocol. This could allow an attacker to read arbitrary local files (LFI) via `file://` protocols, or hit internal services (SSRF).
**Learning:** `fetch()` natively supports various schemes depending on the environment (especially in runtimes like Bun/Node/Deno). Protocol validation must be strict and explicit. We cannot rely solely on the native URL constructor for scheme restrictions since `fetch` natively strips leading whitespaces, creating bypasses for naive regexes.
**Prevention:** Always use `new URL()` to parse the user-provided URL and check the parsed `.protocol` property against an allowlist (e.g. `http:`, `https:`, `data:`, `blob:`). Make sure this check is outside the `try` block or explicitly re-thrown so it doesn't get masked as a generic 'Invalid URL format' error.

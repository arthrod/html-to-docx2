## 2024-05-24 - SSRF and LFI vulnerabilities via unsanitized fetch URLs
**Vulnerability:** The image fetching utilities (`downloadImageToBase64` in multiple files) were passing user-controlled URLs directly to `fetch()` without verifying the protocol schema. This allowed arbitrary Local File Inclusion (LFI) via the `file://` protocol and Server-Side Request Forgery (SSRF) to internal network addresses.
**Learning:** The native `fetch()` API in JS runtime environments (like Node.js, Bun, Deno) will happily resolve and fetch non-HTTP protocols like `file://` if not explicitly prevented. Furthermore, regex-based validation of URLs is often insufficient because `fetch` may strip leading whitespace or handle obscure formats, allowing bypasses.
**Prevention:** Always parse untrusted URLs using the native `URL` constructor (`new URL(String(url).trim(), 'http://dummy.base')`) and explicitly validate that `parsedUrl.protocol` is in a strict allowlist (e.g. `['http:', 'https:', 'data:', 'blob:']`) before passing the string to `fetch()`. The dummy base URL ensures relative paths are safely parsed as HTTP rather than throwing errors and failing open/closed unnecessarily.

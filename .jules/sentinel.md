## 2025-02-12 - Critical Local File Inclusion (LFI) in Bun Environments
**Vulnerability:** Unvalidated absolute URLs in `fetch` calls within image fetching functions (`downloadImageToBase64`, `downloadImage`) allowed Local File Inclusion (LFI) via `file://` protocols, leading to local file reading (e.g. `file:///etc/passwd`).
**Learning:** Bun's native `fetch` API inherently supports `file://` protocols for local file reading. Therefore, what might seem like a simple SSRF risk in a standard Node.js or browser environment turns into a full-blown LFI in Bun when unvalidated user-supplied URLs are fetched directly.
**Prevention:** Always validate URLs explicitly when fetching remote assets. Restrict absolute URLs to safe protocols (`http:`, `https:`, `data:`). Because environments like browsers handle relative or protocol-relative URLs natively with `fetch`, URL parsing must defensively catch `TypeError` to safely allow those non-absolute formats to pass through without failing execution.
